## Vistors' Book Management System

# 2025 경영혁신 Best Practice 경진대회 등록 및 투표권 발급 프로그램 매뉴얼
# 1. 프로그램 개요
     본 프로그램은 '2025 경영혁신 Best Practice 경진대회' 행사의 원활한 진행을 위해 개발되었습니다. 참석자는 온라인으로 간편하게 참석 등록을 하고 고유한 투표권을 발급받을 수 있으며, 관리자는 실시간으로 참석 현황을 모니터링하고 데이터를 관리할 수 있습니다.

# 주요 기능
  * 참석자: 웹 페이지 접속을 통한 정보 등록 및 고유 투표권 발급

  * 관리자: 실시간 참석 현황 대시보드, 참석자 목록 관리, 데이터 백업(CSV), 테스트 기능(시뮬레이션)

# 2. 참석자용 매뉴얼
# 2.1. 접속 및 등록
       행사장에 비치된 QR 코드를 스캔하거나 안내받은 웹 주소로 접속합니다.

       '참석자 등록' 화면에서 아래 정보를 정확히 입력합니다.

         * 소속 (법인명): 드롭다운 목록에서 본인의 소속 법인을 선택합니다.

         * 성명: 이름을 입력합니다.

         * 직급: 드롭다운 목록에서 본인의 직급을 선택합니다.

         * 이메일: 사내 이메일 주소를 입력합니다. (투표권 중복 발급 방지를 위해 사용됩니다.)

      모든 정보 입력 후 '등록 및 투표권 발급' 버튼을 클릭합니다.

      입력한 정보가 올바른지 확인하는 팝업창이 나타납니다. 내용을 다시 한번 확인하고 '최종 제출' 버튼을 누릅니다. (수정이 필요하면 '취소'를 누르고 다시 입력합니다.)

# 2.2. 투표권 확인 및 저장
       최종 제출이 완료되면, 투표권 번호가 표시된 팝업창이 나타납니다.

       예: atec001

       '이미지로 저장' 버튼을 클릭하여 발급된 투표권을 이미지 파일(PNG)로 스마트폰이나 PC에 저장할 수 있습니다.

       '확인' 버튼을 눌러 모든 등록 절차를 마칩니다.

# 3. 관리자용 매뉴얼
# 3.1. 관리자 모드 접속
참석자 등록 페이지 하단의 '관리자 모드' 글씨를 클릭합니다.

로그인 팝업창이 나타나면 아래 관리자 계정 정보를 입력하고 '로그인' 버튼을 클릭합니다.

ID: admin

Password: ***********

# 3.2. 대시보드 기능 소개
  로그인에 성공하면 관리자 대시보드로 이동하며, 실시간 현황을 확인할 수 있습니다.

    * 법인별 참석 현황: 각 법인별 등록 인원을 막대그래프 형태로 시각화하여 보여줍니다. 전체 참석자 대비 법인별 참여율을 직관적으로 파악할 수 있습니다.

    * 참석자 목록: 등록된 모든 참석자의 정보(소속, 성명, 직급, 이메일, 투표권 번호)를 최신순으로 보여주는 표입니다.

    * CSV로 저장: 현재까지 등록된 모든 참석자 목록을 CSV 파일 형식으로 다운로드합니다. 데이터 백업 및 오프라인 분석에 활용할 수 있습니다.

    * 로그아웃: 관리자 모드를 종료하고 초기 등록 화면으로 돌아갑니다.

# 3.3. 데이터 관리 기능
    * 시뮬레이션: 버튼 클릭 시 50명의 가상 참석자 데이터가 자동으로 생성됩니다. 행상 시작 전, 데이터가 차트와 목록에 어떻게 표시되는지 테스트하는 용도로 사용합니다.

    * 시뮬레이션 데이터 삭제: 생성했던 시뮬레이션 데이터만을 식별하여 모두 삭제합니다. 실제 참석자 데이터는 그대로 유지됩니다.

    * 전체 삭제: (주의!) 데이터베이스에 저장된 모든 참석자 정보(실제, 시뮬레이션 포함)를 영구적으로 삭제합니다. 시스템 테스트 완료 후 실제 행사 시작 전에 데이터를 초기화할 때 사용하며, 되돌릴 수 없으므로 신중하게 사용해야 합니다.

# 4. (고급) 시스템 설정 및 배포 매뉴얼
     본 프로그램은 Supabase 데이터베이스와 연동되며, 정적 웹 호스팅을 통해 배포됩니다.

# 4.1. Supabase 데이터베이스 설정
       프로젝트 및 테이블 생성

       Supabase에 가입하고 새 프로젝트를 생성합니다.

       Table Editor 메뉴에서 attendees 이름으로 테이블을 생성하고, 아래 명세에 따라 컬럼을 추가합니다.
       | 컬럼명 | 타입 | 설명 |
       |---|---|---|
       | id | int8 | 기본 키 (Primary Key) |
       | created_at | timestamptz | 데이터 생성 시각 (기본값: now()) |
       | affiliation | text | 소속 |
       | name | text | 성명 |
       | position | text | 직급 |
       | email | text | 이메일 (Unique 옵션 활성화 권장) |
       | ticket | text | 투표권 (Unique 옵션 활성화 권장) |

      * API 키 연동

         Project Settings > API 메뉴에서 Project URL과 anon public 키를 복사합니다.

      다운로드한 event_manager_supabase.html 파일의 <script> 태그 내 SUPABASE_URL과 SUPABASE_ANON_KEY 변수 값을 복사한 정보로 교체합니다.

      * 보안 정책(RLS) 설정

       프로그램이 정상적으로 데이터를 읽고 쓰려면, Supabase의 RLS(Row Level Security) 설정을 비활성화해야 합니다.

       Table Editor에서 attendees 테이블을 선택한 후, 페이지 상단의 'RLS Enabled' 토글 버튼을 클릭하여 비활성화합니다.

# 4.2. GitHub Pages 배포
       저장소(Repository) 생성 및 파일 업로드

       GitHub에 새로운 저장소를 생성합니다.

       수정한 event_manager_supabase.html 파일의 이름을 **index.html**로 변경하여 저장소에 업로드합니다.

       * GitHub Pages 활성화

         저장소의 Settings > Pages 메뉴로 이동합니다.

         Branch를 main (혹은 master)으로 설정하고 Save 버튼을 누릅니다.

      잠시 후, 페이지 상단에 웹사이트 주소가 생성되며, 해당 주소를 통해 누구나 프로그램에 접속할 수 있습니다.
